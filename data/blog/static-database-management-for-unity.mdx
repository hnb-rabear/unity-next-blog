## 1. Static Database

Static Database trong phát triển game là một giải pháp lưu trữ các dữ liệu tĩnh vd như kỹ năng, vật phẩm, nhân vật mà không thay đổi trong suốt quá trình chơi game. Thay vì đọc dữ liệu từ nguồn động (Dynamic Database) chúng ta sử dụng các phương tiện như ScriptableObject, CSV, hoặc JSON để lưu trữ và quản lý thông tin. Việc này giúp cải thiện hiệu suất, dễ quản lý, và tiện lợi trong quá trình phát triển game.

## 2. Thách thức khi Static Database trở nên quá lớn

Trong quá trình phát triển game, chúng ta thường xuyên đối mặt với những thay đổi, nhiều khi cần mở rộng hoặc thay đổi thiết kế, việc phải điều chỉnh tính năng và cấu trúc cơ sở dữ liệu là không thể tránh khỏi. Sử dụng những phương pháp thông thường như ScriptableObject, CSV hoặc JSON để quản lý Static Database có thể không đủ đáp ứng khi chúng ta đối mặt với hệ thống lớn, như trong các game Mid-core trở lên. Quản lý dữ liệu khổng lồ này mà không có công cụ hỗ trợ là một sự phiền toái.

## 3. Excel to Unity - Giải pháp toàn diện để xây dựng và quản lý Static Database

Đây là công cụ linh hoạt và mạnh mẽ, có khẳ năng đáp ứng đầy đủ cho Coder và Designer trong việc thiết kế và quản lý dữ liệu. Nó giải quyết được hầu hết các tình huống liên quan đến thiết kế và quản lý Static Database trong quá trình phát triển game.

Các chức năng bao gồm:

- Database được quản lý hoàn toàn bằng Excel
- Database được export thành dạng text files để để tích hợp dễ dàng vào Unity.
- Biểu diễn JSON data theo dạng bảng linh hoạt và có thể tùy chỉnh đa dạng.
- Quản lý IDs và Constants một cách hiệu quả, dễ dàng điều chỉnh hàng loạt mà không ảnh hưởng đến Database.
- Quản lý việc đa ngôn ngữ thông qua hệ thống Localization, Dễ dàng thêm/sửa, linh hoạt tùy biến với sự hỗ trợ mạnh mẽ từ nhiều Component và API cho Unity.
- Và còn nhiều tính năng khác mang.

## 6. Hướng dẫn sử dụng

### 6.1. Settings

![Settings](/media/excel-2-unity-tab-3.png)

- Json Data Output: Thư mục json data được export vào
- Constants Output: Thư mục IDs, Constants, Localization Component và Localization API được export vào
- Localization Ouput: Thư mục Localization text data được export vào
- Namespace: thiết lập namespace cho các file .cs mà được export
- Separate IDs:
  - TRUE: export các sheet [\*\_IDs] ra thành các file .cs riêng biệt với tên file theo cấu trúc [Tên file excel + _IDs].cs
  - FALSE: tổng hợp tất cả các sheet [\*\_IDs] từ tất cả các file excel thành một rồi export thành một file .cs duy nhất tên là IDs.cs
- Separate Constants:
  - TRUE: export các sheet [\*\_Constants] ra thành các file .cs riêng biệt với tên file theo cấu trúc [Tên file excel + _Constants].cs
  - FALSE: tổng hợp tất cả các sheet [\*\_Constants] từ tất cả các file excel thành một rồi export thành một file .cs duy nhất tên là Constants.cs
- Separate Localization:
  - TRUE: export các sheet [\*\_Localization\_\*] ra thành các nhóm riêng biệt, một nhóm là gồm các file text data, Unity Component và API
  - FALSE: tổng hợp tất cả các sheet [\*\_Localization\_\*] từ tất cả các file excel thành một rồi export thành một nhóm duy nhất
